### Sequence diagram
```plantuml
@startuml
participant User

User -> A: DoWork
activate A

A -> A:  Call Self

A -> B: << createRequest >>
activate B

B -> C: DoWork
activate C
C --> B: WorkDone
destroy C

B --> A: RequestCreated
deactivate B

A -> User: Done
deactivate A

@enduml
```

* Step into function call
  * `->`{.code}
* Step out/return from call
  * `-->`{.code}
