    2 Auth rules

    • @auth directive means accessing that Todo product type will require cognito sign-in
      • OBSERVE: the amplify generated Todo also included owner
        • On Creation, owner is the cognito user
        • On Query, the Todo’s returned will only be the ones created by the owner
    type Todo @model @auth(rules: [{allow: owner}]){
  todoId: ID! @primaryKey
  content: String
}
    type Todo @aws_cognito_user_pools {
	todoId: ID!
	content: String
	createdAt: AWSDateTime!
	updatedAt: AWSDateTime!
	owner: String
}
